# Code of Conduct

## Our Commitment

Sovereign AI SOC is intended to be a professional, respectful, and technically rigorous project.

We are committed to maintaining a collaborative environment where contributors can participate without harassment, discrimination, intimidation, or personal attacks.

## Expected Behavior

Contributors are expected to:

- Communicate respectfully and constructively.
- Focus discussions on the technical issue, not on individuals.
- Give and receive feedback professionally.
- Be transparent about risks, limitations, and assumptions.
- Respect different levels of experience and background.
- Avoid introducing changes that weaken security, privacy, or data-sovereignty principles.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, abuse, threats, or personal attacks.
- Discriminatory language or conduct.
- Public disclosure of private or sensitive information.
- Deliberate introduction of insecure code.
- Sharing credentials, secrets, tokens, logs, or customer-sensitive data in issues, pull requests, or discussions.
- Repeatedly ignoring maintainer guidance or project scope.

## Security and Sensitive Information

Do not post:

- Passwords
- API keys
- Access tokens
- Private certificates
- Customer data
- Internal SOC logs containing sensitive information
- Security vulnerabilities with exploit details in public issues

Security vulnerabilities should be reported according to the process described in `SECURITY.md`.

## Enforcement

Maintainers may take appropriate action when this Code of Conduct is violated, including:

- Requesting edits to comments, issues, or pull requests
- Closing discussions
- Blocking contributors
- Removing inappropriate content
- Rejecting contributions that introduce unacceptable risk

## Scope

This Code of Conduct applies to all project spaces, including:

- Issues
- Pull requests
- Discussions
- Documentation
- Project-related communication channels
