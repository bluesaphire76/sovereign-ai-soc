# v0.4 — Regression and Smoke Validation

## Purpose

This document defines the lightweight pre-release validation workflow for the Sovereign AI SOC local runtime.

The goal is to catch regressions before commit, merge and release without introducing a heavy testing framework too early.

## Scope

The validation covers:

- Python module imports
- Database connectivity
- Required database tables
- Core table row counts
- Public `/health` endpoint
- Authentication protection for selected protected endpoints
- DB retention dry-run
- Frontend production build
- Basic service runtime checks

## Smoke validation script

Script:

```bash
scripts/v0_4_smoke_validation.py
```

Default execution:

```bash
cd ~/lab/ai-soc-assistant
source .venv/bin/activate

python scripts/v0_4_smoke_validation.py
```

The script writes a JSON report under:

```text
reports/validation/
```

Generated JSON reports are ignored by Git.

## Optional authenticated validation

Some endpoints require authentication.

If a valid JWT token is available, pass it through:

```bash
SMOKE_AUTH_TOKEN="<jwt-token>" python scripts/v0_4_smoke_validation.py
```

When no token is provided, the script verifies that selected protected endpoints reject unauthenticated requests.

## Backend static validation

Run:

```bash
python -m py_compile \
  api.py \
  ai_soc_worker.py \
  platform_health.py \
  models.py \
  database.py \
  scripts/v0_4_smoke_validation.py \
  scripts/db_retention_cleanup.py
```

For broader validation:

```bash
python -m compileall . \
  -x '(^|/)(.venv|node_modules|.next|backups|reports)(/|$)'
```

## Frontend build validation

Run:

```bash
cd ~/lab/ai-soc-assistant/frontend
npm run build
```

## Runtime service checks

Run:

```bash
sudo systemctl status ai-soc-api --no-pager
sudo systemctl status ai-soc-frontend --no-pager
sudo systemctl status ai-soc-worker --no-pager
```

Review logs:

```bash
sudo journalctl -u ai-soc-api -n 80 --no-pager
sudo journalctl -u ai-soc-frontend -n 80 --no-pager
sudo journalctl -u ai-soc-worker -n 80 --no-pager
```

## Security checks

Minimum manual checks:

```bash
curl -i http://127.0.0.1:8008/health
curl -i http://127.0.0.1:8008/platform/health
```

Expected behavior:

- `/health` returns HTTP 200
- `/platform/health` rejects unauthenticated requests

For Nginx-exposed runtime, verify security headers from the public/local endpoint:

```bash
curl -I https://<soc-domain>/
```

Expected headers include, depending on endpoint:

- `Content-Security-Policy`
- `X-Frame-Options`
- `X-Content-Type-Options`
- `Referrer-Policy`
- no framework leakage such as `X-Powered-By`

## Pre-commit checklist

Before commit:

```bash
git status --short
python -m py_compile scripts/v0_4_smoke_validation.py scripts/db_retention_cleanup.py
python scripts/v0_4_smoke_validation.py
cd frontend && npm run build
```

Before merge to `main`:

```bash
git status --short
git diff --stat
```

After merge and push:

```bash
sudo systemctl restart ai-soc-api
sudo systemctl restart ai-soc-frontend
sudo systemctl restart ai-soc-worker

python scripts/v0_4_smoke_validation.py
```

## Git hygiene

Do not commit:

- database backups
- generated smoke reports
- generated validation PDFs
- runtime `.env` files
- frontend `.env.local`
- `.next`
- `node_modules`
