# Sovereign AI SOC v0.3 Release Notes

## Release theme

v0.3 focuses on turning Sovereign AI SOC from a working local SOC prototype into a more secure, role-aware and operationally controlled platform.

The release improves access control, auditability, runtime security, operational health monitoring, PostgreSQL runtime maturity, Wazuh ingestion robustness and the overall enterprise UX of the SOC console.

## Major capabilities

### Role-based access control

- Added role-aware access for `ADMIN`, `ANALYST` and `VIEWER`.
- Restricted privileged SOC operations to authorized roles.
- Aligned frontend authorization with the backend session using `/auth/me`.
- Reduced stale-role risks from browser-side stored state.

### User management

- Admin-only full user management.
- Self-service user profile mode for non-admin users.
- Password reset support.
- Admin-only create, update, disable, enable and delete user capabilities.
- Protection against non-admin role changes.
- Protection against self-disable for admin accounts.

### Security audit

- Added security-relevant audit logging.
- Added Admin Security Audit UI.
- Captured login success/failure, RBAC denied events, user management actions and privileged SOC actions.
- Security Audit page is visible only to `ADMIN` users.

### Session and token robustness

- Added improved JWT/session expiry handling.
- Stored token expiry metadata.
- Aligned frontend cookie max-age with backend token validity.
- Added automatic cleanup on expired/invalid session.
- Redirect to login on session expiry.

### Nginx and security headers

- Added security headers through Nginx.
- Disabled server token disclosure.
- Added no-store cache policy for protected/API content.
- Hid `X-Powered-By`.
- Hardened frontend/backend runtime binding to localhost.
- Added versioned deployment references under `deploy/`.

### Secrets and configuration hardening

- Removed weak local-auth defaults.
- Added `.env.example` and frontend env examples.
- Kept runtime secrets out of version control.
- Documented production configuration expectations.

### Enterprise UX improvements

- Moved navigation from top bar to left sidebar on desktop.
- Improved Case Queue layout and controls.
- Improved Case Detail command center and AI analysis presentation.
- Added enterprise-style Incidents page.
- Added Incident Detail action to create or open a case from an incident.
- Improved Users page with explicit admin-only enable/disable actions.

### PostgreSQL 18.4 runtime upgrade

- Migrated PostgreSQL runtime from Docker `postgres:16` to `postgres:18.4`.
- Preserved PostgreSQL 16 rollback container/volume.
- Updated Docker volume mount for PostgreSQL 18+ to `/var/lib/postgresql`.
- Validated API/worker/database connectivity after migration.

### Wazuh ingest and worker hardening

- Recovered ingest after PostgreSQL migration by resetting stale Wazuh watermark.
- Reduced ingest overlap window.
- Tuned worker batch size for local LLM processing.
- Added worker heartbeat updates during polling and alert processing.
- Added worker health stale threshold.
- Added incremental Wazuh watermark progress updates during long batches.
- Added max catch-up guard to avoid infinite historical backlog processing.

### Health observability

- Added Health components for:
  - Event processing queue
  - Active event sources
  - Latest incident freshness
  - Cloudflare tunnel
- Improved visibility of Wazuh ingest, worker state and operational backlog.
- Added Cloudflare tunnel status when remote access is configured.

## Runtime components validated

- FastAPI backend
- Next.js frontend
- PostgreSQL 18.4
- Wazuh Indexer
- Wazuh ingest worker
- Ollama local model runtime
- Qdrant
- Nginx reverse proxy
- Cloudflare tunnel
- RBAC-protected frontend and API routes

## Known design choices

- AI supports triage, correlation, analysis and remediation planning.
- AI does not execute remediation actions.
- Human-in-the-loop remains required for security operations.
- Raw Wazuh events remain available in Wazuh.
- The SOC worker processes recent operational events and avoids unbounded historical catch-up.
