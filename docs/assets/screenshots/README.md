# Screenshot Capture Checklist

This directory is reserved for real product screenshots. Do not add fabricated screenshots.

## Rules

- Use real product screens only.
- Use clean demo data.
- Avoid exposing secrets, tokens, credentials, private hostnames or private IPs unless intentionally part of a controlled local demo.
- Use a consistent browser size and dark theme.
- Prefer enterprise-looking views with meaningful data loaded.
- Review every screenshot before committing.

## Suggested Capture Flow

1. Confirm `/health` is clean.
2. Use a demo account with appropriate permissions.
3. Load each page and wait for data.
4. Hide browser bookmarks and private extensions.
5. Capture at a consistent desktop size, for example 1440x1000.
6. Review for sensitive values.
7. Commit only approved screenshots.

The main README intentionally does not reference screenshot image files until real screenshots are committed.
