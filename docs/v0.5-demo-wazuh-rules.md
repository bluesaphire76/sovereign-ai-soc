# v0.5 — Demo Wazuh Rules

This document describes the Wazuh custom rules used to classify Sovereign AI SOC v0.5 demo scenario pack events.

The rules are designed for controlled, defensive, lab-only demo events emitted by:

```text
tools/synthetic_scenarios/emit_demo_scenario_pack.py
```

## Purpose

The demo scenario pack writes synthetic JSONL events with:

```json
{
  "source": "sovereign-ai-soc-demo-pack",
  "demo": true,
  "synthetic": true,
  "version": "v0.5"
}
```

Without custom Wazuh rules, these events may be classified by generic Wazuh rules such as low-level syslog/authentication rules.

The v0.5 demo rules make demo behavior more predictable:

| Scenario | Rule ID | Level | Expected behavior |
|---|---:|---:|---|
| brute_force_ssh | 100210 | 10 | High-signal incident candidate |
| sudo_escalation | 100211 | 12 | Critical/high-signal incident candidate |
| suspicious_package_activity | 100212 | 8 | Triage/correlation candidate |
| noisy_operational_baseline | 100213 | 3 | Operational noise / observe-only |
| false_positive | 100214 | 4 | False-positive / observe-only context |
| real_incident | 100215 | 13 | Strong incident candidate |
| case_ready | 100216 | 12 | Case-ready incident/correlation candidate |

## Versioned rule file

```text
deploy/wazuh/rules/ai_soc_demo_rules_v0.5.xml
```

## Runtime installation

Copy the rule file into the Wazuh manager container:

```bash
docker cp deploy/wazuh/rules/ai_soc_demo_rules_v0.5.xml \
  single-node-wazuh.manager-1:/var/ossec/etc/rules/ai_soc_demo_rules_v0.5.xml
```

Set ownership and permissions:

```bash
docker exec single-node-wazuh.manager-1 sh -lc \
"chown root:wazuh /var/ossec/etc/rules/ai_soc_demo_rules_v0.5.xml && chmod 640 /var/ossec/etc/rules/ai_soc_demo_rules_v0.5.xml"
```

Restart the Wazuh manager container:

```bash
docker restart single-node-wazuh.manager-1
```

## Validation

Emit one demo event:

```bash
sudo .venv/bin/python tools/synthetic_scenarios/emit_demo_scenario_pack.py \
  --scenario brute_force_ssh \
  --count 1 \
  --host demo-soc-linux-01 \
  --created-by demo_operator
```

Check Wazuh manager alerts:

```bash
docker exec single-node-wazuh.manager-1 sh -lc \
"grep -Rni 'AI SOC demo scenario: brute force SSH\\|100210\\|brute_force_ssh' \
/var/ossec/logs/alerts/alerts.json \
/var/ossec/logs/alerts/alerts.log \
2>/dev/null | tail -n 30"
```

Expected result:

- Wazuh rule id `100210`
- Wazuh level `10`
- Scenario `brute_force_ssh`
- Source `sovereign-ai-soc-demo-pack`

## Safety

These rules do not perform active response and do not execute commands. They only classify lab-only synthetic JSON events.
