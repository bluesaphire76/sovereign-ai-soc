# v0.5 DNS Telemetry Pilot

## Scope

This pilot enables endpoint-based DNS telemetry for Sovereign AI SOC when network SPAN/TAP is not available.

The initial pilot host is `atomicstar`, an Ubuntu host already enrolled as a Wazuh agent.

## Architecture

```text
atomicstar
  -> local DNS collector
  -> /var/log/ai-soc/dns-telemetry.log
  -> Wazuh agent localfile JSON collection
  -> Wazuh manager rule ai_soc_dns_telemetry
  -> AI SOC raw_events
```

## Design principles

- No router or DHCP change
- No switch mirroring required
- No automatic incident creation
- No blocking or remediation
- Read-only telemetry source
- Human-in-the-loop investigation evidence

## Runtime files on atomicstar

```text
/usr/local/bin/ai-soc-dns-collector.py
/etc/systemd/system/ai-soc-dns-collector.service
/var/log/ai-soc/dns-telemetry.log
```

## Wazuh manager rule

```text
/var/ossec/etc/rules/ai_soc_dns_telemetry_rules.xml
```

## Validation

Generate DNS queries on atomicstar:

```bash
getent hosts wikipedia.org
getent hosts cloudflare.com
getent hosts ubuntu.com
```

Check local log:

```bash
sudo tail -n 20 /var/log/ai-soc/dns-telemetry.log
```

Check Wazuh alerts:

```bash
docker exec single-node-wazuh.manager-1 bash -lc \
'tail -n 300 /var/ossec/logs/alerts/alerts.json | grep -i "ai_soc_dns"'
```

Check AI SOC raw events:

```bash
PYTHONPATH=. python - <<'PY'
from sqlalchemy import text
from database import engine

with engine.begin() as conn:
    rows = conn.execute(text("""
        select id, event_timestamp, agent, rule_description
        from raw_events
        where payload_json::text ilike '%ai_soc_dns_query%'
           or rule_description ilike '%AI SOC DNS telemetry%'
        order by id desc
        limit 10
    """)).fetchall()

for row in rows:
    print(row)
PY
```

## Outcome

DNS telemetry is now available as endpoint-based Wazuh evidence and can be normalized into a dedicated `dns_events` table in the next step.
