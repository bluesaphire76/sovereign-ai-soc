# v0.3 Nginx and Security Headers Hardening

## Scope

This hardening step covers the reverse proxy and local service exposure for the Sovereign AI SOC console.

## Implemented controls

- Nginx version/build disclosure reduced with `server_tokens off`.
- Common security headers added:
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: DENY`
  - `Referrer-Policy: no-referrer`
  - `Permissions-Policy`
  - `Cross-Origin-Opener-Policy`
  - `Cross-Origin-Resource-Policy`
  - baseline `Content-Security-Policy`
- HSTS added for the Cloudflare/public hostname path.
- `X-Powered-By` hidden from proxied Next.js responses.
- Protected application pages and API routes forced to `Cache-Control: no-store`.
- Next.js static assets kept cacheable under `/_next/static/`.
- `/api/auth/` explicitly routed to Next.js to avoid conflicts with backend API aliases.
- Backend and frontend services hardened to bind to `127.0.0.1` only.

## Runtime files

Reference copies are stored under:

- `deploy/nginx/`
- `deploy/systemd/`

The active runtime files are still managed under:

- `/etc/nginx/`
- `/etc/systemd/system/`
