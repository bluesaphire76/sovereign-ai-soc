# v0.5 — Demo Scenario Pack

The v0.5 demo scenario pack provides controlled, defensive, lab-only events for Sovereign AI SOC demos.

The goal is to make demos repeatable and understandable without depending on random Wazuh events generated by the environment at that moment.

## Scope

The scenario pack covers:

- brute force SSH
- sudo escalation
- suspicious package activity
- noisy operational baseline
- false positive scenario
- real incident scenario
- case-ready scenario

## Safety model

The scenario pack does not execute attacks, scans, exploitation, credential guessing, malware, persistence, privilege escalation, or system changes.

It only emits synthetic JSONL events that can be collected by Wazuh.

## Output path

Default output:

```text
/var/log/ai-soc-synthetic/alerts.jsonl
```

## Usage

List scenarios:

```bash
python tools/synthetic_scenarios/emit_demo_scenario_pack.py --list
```

Dry-run a single scenario:

```bash
python tools/synthetic_scenarios/emit_demo_scenario_pack.py \
  --scenario brute_force_ssh \
  --count 3 \
  --dry-run
```

Dry-run all scenarios:

```bash
python tools/synthetic_scenarios/emit_demo_scenario_pack.py \
  --all \
  --count 2 \
  --dry-run
```

Emit all scenarios to the default Wazuh-collected JSONL file:

```bash
sudo .venv/bin/python tools/synthetic_scenarios/emit_demo_scenario_pack.py \
  --all \
  --count 5 \
  --host demo-soc-linux-01 \
  --created-by demo_operator
```

## Scenario design

All events are explicitly marked as:

```json
{
  "lab": true,
  "demo": true,
  "synthetic": true,
  "version": "v0.5"
}
```

This makes demo data clearly distinguishable from runtime security data.

## Integration note

This step intentionally does not modify:

- `api.py`
- frontend pages
- RBAC policy
- `/synthetic-tests/*` endpoints

The scenario pack is file-based and designed to work with the existing Wazuh collection path.
