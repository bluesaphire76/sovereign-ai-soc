# AI SOC v0.7 Expanded Validation Harness

This document describes the v0.7 validation harness used for local, CI, and
demo readiness checks.

## Scope

The harness is implemented in `scripts/v0_7_validation_harness.py` and validates:

- repository shape and branch hygiene
- Python compile health and core backend imports
- optional database connectivity and v0.7 table coverage
- API health, auth/RBAC behavior, Detection Control inventory, service operations, incident timeline, investigation graph, and report/export endpoints when a live API and credentials are provided
- demo scenario pack availability and synthetic JSONL generation
- observability deployment artifacts and optional Grafana/Prometheus/Alertmanager reachability
- security guardrails for tracked secret files, generated validation reports, `api.py` churn, and service restart command handling

The default mode is non-destructive. It does not apply detection configuration,
roll back configuration, create lifecycle items, restart services, or execute
remediation actions.

## Commands

Local validation:

```bash
python3 scripts/v0_7_validation_harness.py --mode local --output-dir reports/validation --no-destructive
```

CI validation without live runtime:

```bash
python3 scripts/v0_7_validation_harness.py --mode ci --output-dir reports/validation --no-destructive
```

Demo validation with runtime credentials:

```bash
python3 scripts/v0_7_validation_harness.py \
  --mode demo \
  --base-url https://YOUR_HOST \
  --frontend-url https://YOUR_HOST \
  --token "$AI_SOC_API_TOKEN" \
  --output-dir reports/validation \
  --no-destructive
```

When no token or username/password is provided, authenticated checks are marked
as `SKIP` with the message `SKIPPED_AUTH_REQUIRED`.

## Reports

The harness writes:

- `reports/validation/v0.7-validation-summary.json`
- `reports/validation/v0.7-validation-summary.md`
- `reports/validation/v0.7-validation-details.json`

Generated reports are ignored by git. The JSON details file redacts secret-like
keys and stores only whether credentials were provided, never their values.

## Exit Codes

- `0`: no failed checks
- `1`: at least one failed check

Warnings are intentionally non-blocking in local/demo mode for optional services
such as Grafana, Prometheus, Alertmanager, or a temporarily unavailable live API.
Use `--strict` to escalate high-severity warnings to failures.

## Notes

This repository currently keeps backend modules at the repository root rather
than under a `backend/` directory, so compile/import checks are aligned to the
existing layout.
