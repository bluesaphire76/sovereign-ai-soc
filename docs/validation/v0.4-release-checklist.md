# Sovereign AI SOC v0.4 Release Checklist

## Scope

Release v0.4 focuses on operational maturity:

- event aggregation and deduplication;
- raw events / security alerts / incidents / cases separation;
- correlation-first ingestion;
- deterministic noise suppression;
- worker/backlog metrics;
- AI triage hardening and fallback;
- risk normalization and controlled historical backfill;
- AI runtime observability;
- enterprise report templates;
- DB retention and backup/restore documentation;
- regression/smoke validation;
- production/demo hardening documentation.

## Git hygiene

- [ ] Working tree is clean before release branch.
- [ ] No `.env` files are tracked.
- [ ] No frontend `.env.local` is tracked.
- [ ] No DB backups are tracked.
- [ ] No generated smoke reports are tracked.
- [ ] No generated PDFs are tracked.
- [ ] `.env.example` is up to date.
- [ ] `frontend/.env.local.example` is up to date.

Commands:

```bash
git status --short
git ls-files | grep -Ei '(^|/)\.env|\.env\.local|secret|credential|token|key|pem|p12|pfx' || true
```

## Database

- [ ] PostgreSQL is running.
- [ ] PostgreSQL backup exists before release.
- [ ] Core table counts are reviewed.
- [ ] Retention script dry-run returns expected result.
- [ ] No destructive cleanup has been run without backup.

Commands:

```bash
docker ps --format "table {{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}"

docker exec -i postgres-soc psql -U soc -d aisoc <<'SQL'
select version();

select
  'raw_events' as table_name, count(*) from raw_events
union all
select 'security_alerts', count(*) from security_alerts
union all
select 'event_aggregates', count(*) from event_aggregates
union all
select 'incidents', count(*) from incidents
union all
select 'incident_cases', count(*) from incident_cases
union all
select 'security_audit_events', count(*) from security_audit_events;
SQL
```

Backup:

```bash
mkdir -p backups

docker exec postgres-soc pg_dump -U soc aisoc \
  > "backups/ai_soc_before_v0.4_release_$(date -u +%Y%m%dT%H%M%SZ).sql"

latest_backup=$(ls -1t backups/ai_soc_before_v0.4_release_*.sql | head -1)

echo "$latest_backup"
ls -lh "$latest_backup"
head -5 "$latest_backup"
```

## Final release decision

Release can proceed only if:

- smoke validation has zero failures;
- frontend build succeeds;
- runtime services are active;
- no runtime secrets are tracked;
- public access is gated;
- DB backup exists;
- known exposed lab services are documented and consciously accepted or restricted.
