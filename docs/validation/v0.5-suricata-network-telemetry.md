# v0.5 Suricata Network Telemetry Integration

## Purpose

This integration extends Sovereign AI SOC from host-centric Wazuh telemetry to multi-source security visibility by adding Suricata network telemetry.

The goal is not to generate more raw alerts, but to provide enterprise-grade investigation context through correlation-driven network evidence.

## Architecture

```text
Suricata Docker sensor
  -> EVE JSON
  -> Suricata ingest worker
  -> network_events table
  -> read-only Network Events API
  -> Network Activity UI
  -> Incident Network Evidence panel
  -> Health observability
```

## Components

### Suricata Docker sensor

Runtime deployment:

```text
deploy/suricata/docker-compose.yml
```

Container name:

```text
ai-soc-suricata
```

The sensor runs in IDS mode and writes EVE JSON to:

```text
deploy/suricata/logs/eve.json
```

### Network events table

Network telemetry is persisted in:

```text
network_events
```

Supported event types:

- alert
- dns
- http
- tls
- flow

Events are deduplicated by `event_fingerprint`.

### Ingest worker

Worker:

```text
workers/suricata_ingest_worker.py
```

Systemd service:

```text
ai-soc-suricata-ingest
```

Versioned unit file:

```text
deploy/systemd/ai-soc-suricata-ingest.service
```

The worker uses byte offset state stored in:

```text
suricata_ingest_state
```

### Time alignment

The integration uses UTC-normalized timestamps.

- PostgreSQL stores `TIMESTAMPTZ`
- Suricata EVE JSON includes timezone-aware timestamps
- the systemd worker runs with `TZ=UTC`
- UI rendering remains responsible for local display formatting

Validation helper:

```text
scripts/validate_time_alignment.py
```

### API

Read-only network telemetry endpoints:

```text
GET /network-events
GET /network-events/recent
GET /network-events/summary
GET /incidents/{incident_id}/network-evidence
```

RBAC is read-only and available to authenticated roles:

- ADMIN
- ANALYST
- VIEWER

### Frontend

Network activity page:

```text
/network-events
```

Incident detail enrichment:

```text
Network Evidence panel
```

The Incident Detail panel is read-only and positioned below the Structured context / Audit trail area without changing the existing UX layout.

### Health observability

The Health page includes:

```text
Suricata sensor
Suricata ingest
Latest Network event freshness
```

These are rendered through the existing Health page component system without layout or styling changes.

### Database

```bash
PYTHONPATH=. python - <<'PY'
from sqlalchemy import text
from database import engine

with engine.begin() as conn:
    total = conn.execute(text("select count(*) from network_events")).scalar()
    by_type = conn.execute(text("""
        select event_type, count(*)
        from network_events
        group by event_type
        order by count(*) desc
    """)).fetchall()
    state = conn.execute(text("""
        select source, file_path, byte_offset, updated_at
        from suricata_ingest_state
        where source = 'suricata'
    """)).fetchone()

print("network_events:", total)
print("by_type:", by_type)
print("state:", state)
PY
```

## Enterprise positioning

This integration strengthens the product by adding network telemetry as investigation evidence, while preserving the existing principles:

- local-first
- data sovereignty
- human-in-the-loop
- correlation-first
- read-only evidence enrichment
- no blind AI execution
- no automatic blocking
- no IPS inline mode
