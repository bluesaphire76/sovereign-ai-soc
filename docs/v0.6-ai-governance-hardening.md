# v0.6 — AI Governance Hardening

Step 13 introduces a shared AI governance foundation for Sovereign AI SOC.

The goal is to standardize how AI-assisted outputs are classified, labeled and reviewed across investigation, detection engineering, remediation planning and executor governance.

## Principles

- Facts require evidence.
- Inferences require rationale.
- Speculation must be labeled.
- Unsupported claims must not be presented as facts.
- Low-confidence output requires human review.
- Fallback-generated output must be visible.
- Simulation and mock/no-op dispatch must not be presented as production execution.
- AI must not bypass human governance.

## Shared Classifications

The governance layer defines four claim classifications:

- `EVIDENCE_BACKED`
- `INFERRED`
- `SPECULATIVE`
- `UNSUPPORTED`

## Safety Labels

The governance layer also defines reusable presentation labels such as:

- `LOW_CONFIDENCE`
- `REQUIRES_HUMAN_REVIEW`
- `SIMULATION_ONLY`
- `EXECUTION_DISABLED`
- `MOCK_ONLY`
- `FALLBACK_GENERATED`

## Current Scope

This step currently adds shared models and deterministic validators only.

It does not yet change runtime investigation, detection engineering, remediation, executor or frontend behavior.

## Non-Goals

This step does not introduce:

- autonomous AI behavior
- remediation execution
- arbitrary command execution
- frontend redesign
- new router logic
- changes to `api.py`
