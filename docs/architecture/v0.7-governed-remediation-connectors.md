# AI SOC v0.7.0 Governed Remediation Connectors

v0.7 introduces a governed remediation proposal layer. It does not introduce
autonomous remediation.

The intended flow is:

```text
AI recommends -> AI SOC creates a governed proposal -> human reviews -> authorized user approves/rejects -> safe internal conversion or proposal-only tracking
```

## Safety Boundary

The governed connector layer does not:

- execute firewall, SOAR, EDR or ticketing API calls;
- restart services;
- apply detection rule lifecycle items;
- disable rules automatically;
- convert free-form AI text into executable payloads;
- run shell commands.

High-risk actions can be documented, reviewed and tracked only. External connector placeholders remain disabled and execution unsupported.

## Action Catalog

The catalog is exposed at:

```text
GET /remediation/catalog/actions
```

Initial action types:

- `CREATE_CASE_ACTION`
- `GENERATE_REMEDIATION_PLAN`
- `GENERATE_CONTAINMENT_CHECKLIST`
- `CREATE_DETECTION_RULE_DRAFT`
- `CREATE_NOISE_SUPPRESSION_DRAFT`
- `CREATE_EXCEPTION_DRAFT`
- `LINK_RECOMMENDED_ACTION_TO_CASE`
- `PREPARE_SERVICE_RESTART`
- `PREPARE_RULE_DISABLE`
- `PREPARE_IP_BLOCK`
- `PREPARE_EXTERNAL_TICKET`
- `PREPARE_SOAR_PLAYBOOK`

Low-risk/internal actions can create internal review objects. Medium/high-risk and external actions require admin approval and remain draft/proposal-only when appropriate.

## Connector Catalog

The connector catalog is exposed at:

```text
GET /remediation/catalog/connectors
```

Connector types:

- `CASE_MANAGEMENT`
- `DETECTION_CONTROL`
- `REPORT_GENERATOR`
- `SERVICE_OPERATIONS`
- `TICKETING_PLACEHOLDER`
- `FIREWALL_PLACEHOLDER`
- `SOAR_PLACEHOLDER`
- `EDR_PLACEHOLDER`

`CASE_MANAGEMENT`, `DETECTION_CONTROL` and `REPORT_GENERATOR` can create internal records/drafts. `SERVICE_OPERATIONS` only links restart intent to the existing operational workflow. External placeholders are disabled and proposal-only.


Initial templates:

- `CASE_CLOSURE_READINESS`

Creating from a playbook creates a governed proposal. It does not execute playbook actions.

## Proposal Lifecycle

Persistent tables:

- `remediation_proposals`
- `remediation_proposal_events`

States:

- `DRAFT`
- `PROPOSED`
- `APPROVED`
- `REJECTED`
- `CANCELLED`
- `CONVERTED`

Server-side transitions are enforced. Invalid transitions return `400`.

## RBAC

Backend RBAC is enforced in the central allowlist and again in the remediation proposal service.

- ADMIN and ANALYST can create drafts and submit proposals.
- ADMIN approves and rejects proposals.
- VIEWER is read-only.
- Medium/high-risk conversion requires ADMIN.
- External connector execution is not permitted for any role.

Denied writes create `REMEDIATION_ACTION_DENIED` security audit events when the request reaches the remediation service.

## Audit

Proposal lifecycle events are stored in `remediation_proposal_events`.

Security audit events include:

- `REMEDIATION_PROPOSAL_CREATED`
- `REMEDIATION_PROPOSAL_UPDATED`
- `REMEDIATION_PROPOSAL_SUBMITTED`
- `REMEDIATION_PROPOSAL_APPROVED`
- `REMEDIATION_PROPOSAL_REJECTED`
- `REMEDIATION_PROPOSAL_CANCELLED`
- `REMEDIATION_PROPOSAL_CONVERTED`
- `REMEDIATION_AI_RECOMMENDATION_CONVERTED`
- `REMEDIATION_PLAYBOOK_PROPOSAL_CREATED`
- `REMEDIATION_CONNECTOR_PLACEHOLDER_USED`
- `REMEDIATION_ACTION_DENIED`

Audit payloads avoid raw sensitive connector payloads.

## Detection Control Integration

Conversions for these actions create Detection Control Plane lifecycle items in `DRAFT` state only:

- `CREATE_DETECTION_RULE_DRAFT`
- `CREATE_NOISE_SUPPRESSION_DRAFT`
- `CREATE_EXCEPTION_DRAFT`

They are not submitted, approved, applied or restarted automatically.

## Service Operations Integration

`PREPARE_SERVICE_RESTART` records a governed restart intent and links to Service Operations. It does not call `systemctl`, Docker, or any restart API.

## AI Recommendation Integration

`POST /remediation/proposals/from-ai-recommendation` normalizes AI recommendation text into a governed proposal. Ambiguous or incomplete high-risk text falls back to a low-risk/document-only proposal. External AI calls are not made by this conversion path.

## Frontend

Incident Detail and Case Detail include a `Governed Remediation` section.

The section supports:

- viewing linked proposals;
- manual proposal creation;
- creating proposals from AI recommendations where structured recommendations are available;
- creating proposals from playbook templates;
- submit/approve/reject/convert actions based on role and state;
- warning labels for high-risk/proposal-only actions;
- links to converted internal objects where available.

## Deployment

For a fresh database, `create_tables.py` creates the new tables through SQLAlchemy metadata.

For an existing deployment, run:

```bash
python scripts/create_remediation_proposal_tables.py
```

Then restart/redeploy API and frontend.

## Manual Validation

Recommended manual flow:

1. Login as ADMIN.
2. Open an incident detail page.
3. Open `Governed Remediation`.
4. Create a manual proposal.
5. Create a proposal from an AI recommendation.
6. Create a proposal from a playbook.
7. Submit and approve a proposal.
8. Convert a low-risk case action proposal.
9. Convert a noise suppression proposal and confirm a DRAFT lifecycle item is created only.
10. Create/approve/convert a service restart proposal and confirm no restart occurs.
11. Create an IP block proposal and confirm it remains proposal-only.
12. Login as ANALYST and verify approval is denied.
13. Login as VIEWER and verify write actions are unavailable/denied.
14. Check proposal history and Security Audit.

## Current Limitations

- External connector credentials and execution are intentionally out of scope.
- Detection draft payloads require concrete match criteria and owner.
- Proposal conversion to case action requires a linked case or a case-linked incident.
- The v0.6 controlled-execution path remains limited to allowlisted internal
  workflow records. Proposal conversion does not widen that boundary.
- Service restart proposals do not execute restarts; Service Operations still
  requires preview, ADMIN confirmation and allowlisted runtime support.
