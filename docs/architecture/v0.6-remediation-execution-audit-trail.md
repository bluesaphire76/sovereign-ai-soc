# v0.6 Step 16 - Execution Audit Trail

Step 16 adds a structured remediation execution audit trail for the AI remediation workflow.

The audit trail is read-only. It records planning, remediation intelligence source, approval gates, dry-run, rollback readiness and the execution boundary. It does not execute remediation or rollback actions.

## Endpoint

```text
GET /incidents/{incident_id}/remediation-audit-trail
```

The router delegates to `remediation.audit_trail`. `api.py` only contains the centralized RBAC allowlist entry.

## Events

The computed trail includes events such as:

- `PLAN_GENERATED`
- `AI_REMEDIATION_INTELLIGENCE_USED`
- `DETERMINISTIC_FALLBACK_USED`
- `APPROVAL_REQUIRED`
- `DRY_RUN_REQUIRED`
- `DRY_RUN_COMPLETED`
- `ROLLBACK_READINESS_REQUIRED`
- `ROLLBACK_READINESS_CHECKED`
- `EXECUTION_NOT_SUPPORTED`
- `HUMAN_VALIDATION_REQUIRED`

If a policy readiness blocker is present, the trail can also include `EXECUTION_BLOCKED_BY_POLICY`.

## Data Model

Each record includes:

- event id
- event type
- timestamp
- actor and actor role
- summary
- decision
- policy status
- evidence references
- rationale
- metadata

The response summary explicitly states:

- `execution_attempted: false`
- `execution_blocked: true`
- `approval_required: true`

## Frontend

Incident Detail includes a compact **Remediation Audit Trail** section in the investigation console. It shows a read-only timeline of governance events and the execution boundary.

The panel is loaded asynchronously and does not block the incident page.

## Boundary

Step 16 does not:

- execute remediation
- execute rollback
- run shell commands
- mutate target systems
- add run/execute buttons
- bypass approval

Human approval remains mandatory before any future operational action.

## Persistence

This step exposes a computed audit trail from current remediation intelligence, dry-run, rollback readiness and readiness metadata. Persistence hardening can be added in a future step if product requirements need durable remediation audit records beyond the existing domain models.
