# AI SOC v0.7.0 - Advanced Incident Timeline

## Scope

Step 6 adds a read-only incident timeline to Incident Detail. The timeline aggregates records that already have a direct incident or case link. It does not create graph views and does not synthesize events.

## Backend Endpoints

- `GET /incidents/{incident_id}/timeline`
- `GET /incidents/{incident_id}/timeline/summary`
- `GET /incidents/{incident_id}/timeline/capabilities`

Supported query filters on `/timeline`:

- `category`, repeated or comma-separated
- `source`
- `severity`
- `time_from`, `time_to`
- `key_only`
- `entity`
- `sort=asc|desc`
- `limit`, default `200`, maximum `500`
- `cursor`, offset cursor for the current filtered set
- `include_raw_payload`, default `false`

## Timeline Sources

The timeline is built from these linked tables only:

- `incidents`
- `raw_events`
- `security_alerts`
- `event_aggregates`
- `incident_audit`
- `incident_notes`
- `incident_cases`
- `case_incidents`
- `case_audit`
- `case_actions`
- `case_ai_analyses`

## Categories

Implemented when linked rows exist:

- `RAW_EVENT`
- `SECURITY_ALERT`
- `AGGREGATED_DUPLICATE`
- `CORRELATION_DECISION`
- `INCIDENT_CREATED`
- `INCIDENT_STATUS_CHANGE`
- `INCIDENT_SEVERITY_CHANGE`
- `AI_ANALYSIS`
- `AI_COMMAND_BRIEF`
- `ANALYST_NOTE`
- `CASE_CREATED`
- `CASE_STATUS_CHANGE`
- `CASE_ACTION_CREATED`
- `CASE_ACTION_COMPLETED`
- `NOISE_SUPPRESSION_MATCH`
- `EXCEPTION_MATCH`
- `UNKNOWN`

Capability notes report unavailable categories such as `DETECTION_RULE_MATCH`, `SERVICE_OPERATION`, and `REPORT_EXPORTED` when the current schema has no incident-linked source.

## RBAC And Raw Payloads

All authenticated roles can read the timeline, summary, and capabilities through the existing API middleware.

Raw payloads are redacted by default. `include_raw_payload=true` is allowed only for `ADMIN` and `ANALYST`. `VIEWER` requests for raw payloads return `403` and write a security audit event. Successful raw payload access is also audited.

The frontend does not request raw payloads.

## Performance Limits

The endpoint uses incident-linked IDs first and avoids unbounded global scans. Output is limited to `200` items by default and `500` maximum. Internal linked-row reads are capped to keep incident detail loads bounded.

## Frontend

Incident Detail now includes `Advanced Incident Timeline` after Evidence & Correlation. It provides:

- summary counters
- view filters for All, Key events, Raw, Alerts, AI, Lifecycle, Case, Notes, Detection/noise
- source, severity, sort, and entity filters
- collapsible event details
- capability notes for unavailable source categories

## Known Limitations

- Case Detail keeps its existing timeline for now.
- Graph view and relationship visualization are intentionally out of scope for Step 6.
- Service operations and report exports are not shown until those records carry an incident foreign key or reliable incident reference.
- Detection rule hit details are shown only when represented by linked security alert status or aggregate status; lifecycle item hit history is not incident-linked yet.
