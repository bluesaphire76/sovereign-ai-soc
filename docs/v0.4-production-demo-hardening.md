# v0.4 — Production / Demo Hardening Notes

## Purpose

This document defines the production-like and remote-demo hardening baseline for Sovereign AI SOC v0.4.

The goal is to make the local-first SOC console safer to operate during demos while preserving the sovereign/local architecture of the project.

## Current runtime posture

Validated runtime components:

| Component | Runtime posture |
|---|---|
| FastAPI backend | Bound to `127.0.0.1:8008` |
| Next.js frontend | Bound to `127.0.0.1:3000` |
| PostgreSQL | Bound to `127.0.0.1:5432` |
| Nginx local HTTPS | Exposes local reverse proxy |
| Cloudflare Access | Protects public demo endpoint |
| Wazuh stack | Runs in Docker |
| Smoke validation | `scripts/v0_4_smoke_validation.py` |

## Positive controls already in place

The following controls are already part of the v0.3/v0.4 hardening baseline:

- Backend runtime bound to loopback.
- Frontend runtime bound to loopback.
- PostgreSQL bound to loopback.
- Nginx reverse proxy in front of backend/frontend.
- Security headers configured through Nginx.
- Cloudflare Access protects the public demo endpoint.
- Runtime `.env` files are not tracked by Git.
- Frontend `.env.local` is not tracked by Git.
- Generated validation reports and DB backups are ignored by Git.
- Smoke validation script exists and can be run before release.
- DB retention cleanup exists and is dry-run by default.
- Backup/restore documentation exists.

## Important exposure note

During local lab operation, some Wazuh/Qdrant services may listen on `0.0.0.0`.

Examples observed during v0.4 validation:

- Wazuh Dashboard on port `443`
- Wazuh Indexer on port `9200`
- Wazuh Manager API on port `55000`
- Wazuh agent ports such as `1514` / `1515`
- Qdrant on ports `6333` / `6334`

This is acceptable only for a controlled lab network.

For a public demo, production-like deployment, or any untrusted network, these ports must be restricted using one or more of:

- host firewall rules;
- Docker bind address changes;
- VPN-only access;
- Cloudflare Tunnel / Access;
- trusted LAN allowlisting;
- private management network.

## Recommended demo exposure model

Preferred public demo model:

```text
Internet
  -> Cloudflare Access
    -> Cloudflare Tunnel / local Nginx
      -> Nginx reverse proxy
        -> Next.js frontend on 127.0.0.1:3000
        -> FastAPI backend on 127.0.0.1:8008
```

Avoid exposing backend, frontend, PostgreSQL, Qdrant, Wazuh Indexer or Wazuh Manager directly to the Internet.

## Minimum pre-demo checklist

Before a remote demo:

```bash
cd ~/lab/ai-soc-assistant
git status --short
```

Expected:

```text
<empty>
```

Validate services:

```bash
sudo systemctl status ai-soc-api --no-pager
sudo systemctl status ai-soc-frontend --no-pager
sudo systemctl status ai-soc-worker --no-pager
```

Validate health:

```bash
curl -s http://127.0.0.1:8008/health | python3 -m json.tool
curl -i http://127.0.0.1:8008/platform/health | sed -n '1,80p'
```

Expected:

- `/health` returns HTTP 200.
- `/platform/health` returns HTTP 401 or 403 without authentication.

Validate smoke:

```bash
source .venv/bin/activate
python scripts/v0_4_smoke_validation.py
```

Expected:

```text
fail = 0
```

Validate exposed ports:

```bash
ss -tulpn | grep -E '(:8008|:3000|:3001|:8443|:80|:443|:5432|:6333|:6334|:9200|:55000|:1514|:1515)' || true
```

Review any service listening on `0.0.0.0`.

Validate Docker exposure:

```bash
docker ps --format "table {{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}"
```

Review any service published to `0.0.0.0`.

Validate Nginx security headers:

```bash
sudo nginx -T 2>/dev/null | grep -nEi \
  "server_name|listen|proxy_pass|add_header|Content-Security|X-Frame|X-Content|Referrer|server_tokens|X-Powered-By" \
  | sed -n '1,260p'
```

Validate public access gate:

```bash
curl -I https://soc.varqon.net/ | sed -n '1,120p'
```

Expected:

- Cloudflare Access redirect or authenticated access challenge.
- No direct unauthenticated application exposure.

## Secrets checklist

Before demo or release:

```bash
git ls-files | grep -Ei '(^|/)\.env|\.env\.local|secret|credential|token|key|pem|p12|pfx' || true
```

Expected tracked files:

```text
.env.example
docs/v0.3-secrets-config-hardening.md
```

Runtime files may exist locally but must not be tracked:

```bash
find . -maxdepth 3 -type f \
  \( -name ".env" -o -name ".env.local" -o -name "*.pem" -o -name "*.key" -o -name "*.p12" -o -name "*.pfx" \) \
  -not -path "./.git/*" \
  -not -path "./.venv/*" \
  -not -path "./frontend/node_modules/*" \
  -print
```

Expected runtime local files:

```text
./.env
./frontend/.env.local
```

They must remain untracked.

## Firewall guidance

For local lab usage, ensure only trusted devices can reach exposed SOC/Wazuh ports.

For demo or production-like usage, prefer:

- backend and frontend on loopback only;
- PostgreSQL on loopback only;
- Qdrant not publicly exposed;
- Wazuh Indexer not publicly exposed;
- Wazuh Manager API not publicly exposed;
- Wazuh agent ports restricted to trusted agents only;
- Cloudflare Access or VPN for remote access.

## Backup before risky operations

Before any destructive test, migration, retention apply, or release hardening change:

```bash
mkdir -p backups

docker exec postgres-soc pg_dump -U soc aisoc \
  > "backups/ai_soc_before_demo_or_release_$(date -u +%Y%m%dT%H%M%SZ).sql"

latest_backup=$(ls -1t backups/ai_soc_before_demo_or_release_*.sql | head -1)

echo "$latest_backup"
ls -lh "$latest_backup"
head -5 "$latest_backup"
```

Expected header:

```text
-- PostgreSQL database dump
```

## Runtime validation after restart

```bash
sudo systemctl restart ai-soc-api
sudo systemctl restart ai-soc-frontend
sudo systemctl restart ai-soc-worker

sudo journalctl -u ai-soc-api -n 80 --no-pager
sudo journalctl -u ai-soc-frontend -n 80 --no-pager
sudo journalctl -u ai-soc-worker -n 80 --no-pager

source .venv/bin/activate
python scripts/v0_4_smoke_validation.py
```

Expected:

- services active;
- no repeated restart loops;
- smoke validation has zero failures.

## Known non-blocking observations

The following observations are acceptable for the current local lab but should be reviewed before production-like exposure:

- Wazuh and Qdrant Docker services may publish ports on `0.0.0.0`.
- Local Nginx HTTPS may listen on `0.0.0.0:8443`.
- Wazuh Dashboard may publish `0.0.0.0:443`.
- Historical frontend startup logs may show a missing `.next` build if the service was started before `npm run build`; current service state must be validated after the latest build.

## Release readiness position

A v0.4 demo is acceptable when:

- Cloudflare Access is active for public access;
- no backend/frontend direct public exposure exists;
- runtime secrets are not tracked;
- smoke validation passes;
- frontend build passes;
- DB backup exists;
- exposed Wazuh/Qdrant ports are restricted or consciously accepted for a trusted lab network only.
